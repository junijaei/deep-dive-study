# 문서 생성 규칙

`week/weekXX_original.md`를 원본으로 삼아 **챕터별 요약 파일**과 **주차 인덱스 파일**을 생성하는 규칙을 정리한다.

---

## 1. 디렉토리 & 파일 구조

```
week/
  weekXX_original.md   ← 원본 (수동 작성)
  weekXX.md            ← 주차 인덱스 (생성)

chapters/
  chNN/
    README.md          ← 챕터 요약 (생성)
    questions.md       ← 문제 모음 (선택, 수동 작성)
```

### 챕터 디렉토리 명명 규칙

- 원본의 `# N. 제목` 섹션 번호를 기준으로 한다.
- 디렉토리명은 `ch` + 두 자리 이상 숫자 (예: `ch01`, `ch09`, `ch10`).

---

## 2. 챕터 파일 생성 규칙 (`chapters/chNN/README.md`)

### 2-1. 챕터 분리

- 원본에서 `# N. 제목` 단위로 내용을 분리한다.
- 챕터 사이의 구분선(`---`)은 **제거**한다.
- `# N. 제목` 헤딩은 챕터 파일에 **그대로 포함**한다.

### 2-2. aside 파싱

원본의 `<aside>` 블록을 GitHub Flavored Markdown 콜아웃으로 변환한다.

**변환 절차**

1. `<aside>` 여는 태그를 제거한다.
2. aside 내부 첫 번째 줄의 이모지(예: `💡`)를 제거한다.
3. aside 내부의 빈 줄을 제거한다.
4. 남은 내용 줄 각각의 앞에 `> `를 붙인다.
5. 변환된 블록의 **최상단**에 `> [!NOTE]` 를 추가한다.
6. `</aside>` 닫는 태그를 제거한다.

**변환 예시**

원본:
```
<aside>
💡

식별자란 변수의 이름. 식별자는 값이 아니라 메모리 주소를 기억하고 있다.
선언에 의해 자바스크립트 엔진에 식별자의 존재를 알린다.

</aside>
```

변환 결과:
```
> [!NOTE]
> 식별자란 변수의 이름. 식별자는 값이 아니라 메모리 주소를 기억하고 있다.
> 선언에 의해 자바스크립트 엔진에 식별자의 존재를 알린다.
```

### 2-3. 그 외 콘텐츠

`<aside>` 블록을 제외한 나머지 콘텐츠(헤딩, 표, 리스트, 코드 블록 등)는 **원본 그대로** 유지한다.

---

## 3. 주차 인덱스 파일 생성 규칙 (`week/weekXX.md`)

아래 구조를 따른다.

```markdown
# Week NN

## 진행 범위
- chA ~ chB

## 요약 문서
- [chNN](../chapters/chNN/README.md)
- ...

## 핵심 요약
- ...

## 문제 모음 (선택)
- [chNN](../chapters/chNN/questions.md)
- ...

## 토론 내용 (선택)
- ...
```

- `# Week NN`: 해당 주차 번호.
- `## 진행 범위`: 원본에 포함된 챕터 범위를 `chA ~ chB` 형식으로 기재.
- `## 요약 문서`: 생성된 챕터 파일 각각에 대한 상대 경로 링크 목록.
- `## 핵심 요약`: 원본의 내용 중 핵심이 되는 내용들만 간단하게 정리.
- `## 문제 모음`: 해당 주차에 questions.md가 있는 챕터의 링크 목록. 문제가 없으면 섹션 자체를 생략한다.
- `## 토론 내용`: 해당 주차 스터디에서 나온 토론 내용을 이름 없이 주제별로 정리. 토론이 없으면 섹션 자체를 생략한다.

---

## 4. 문제 모음 파일 규칙 (`chapters/chNN/questions.md`)

스터디 중 다룬 문제를 챕터 단위로 모아 두는 파일이다. 해당 챕터에 내용이 없으면 생성하지 않는다.

### 4-1. 구조

```markdown
# chNN 질문 모음

## Q1. 질문 내용

<details>
<summary>정답</summary>

정답 내용

</details>

---

## Q2. 질문 내용
...
```

### 4-2. 작성 규칙

- 질문은 `## QN.` 형식으로 번호를 붙인다.
- 정답은 `<details>` 태그로 감싸 기본적으로 접힌 상태로 제공한다.
- 스터디원 개인의 풀이나 이름은 포함하지 않는다.
- 질문이 속하는 챕터를 기준으로 파일을 분리한다 (하나의 questions.md에 여러 챕터 내용을 섞지 않는다).
