# 11. 원시 값과 객체의 비교

> [!NOTE]
> 원시 값은 변경 불가능한 값 (변수에 실제 값 저장)
> 객체 값은 변경 가능한 값 (변수에 참조 값 저장)

## 11.1 원시 값

- 원시 값은 변경 불가능한 값, 즉 읽기 전용인 값이다.
- 원시 값을 할당한 변수에 새로운 원시 값을 재 할당하면 메모리 공간에 저장된 원시 값을 변경하는 게 아니라 새로운 공간을 재할당한 후, 그 주소에 원시 값을 저장한다.
- 불변성을 갖는 원시 값을 할당한 변수는 재할당 이외에 변수 값을 변경할 수 있는 방법이 없다.
- 문자열은 **유사 배열 객체**이면서 **이터러블**이다.
- 문자열 또한 변경 불가능한 값이기에 문자열의 인덱스로 접근해 값을 수정할 수 없다.

### 값에 의한 전달

- 원시 값을 가진 변수를 다른 변수에 할당하면 **값에 의한 전달**이 일어난다.

```jsx
var score = 80;

var copy = score;

console.log(score, copy); // 80, 80
console.log(score === copy); // true

// score와 copy에 할당된 값은 다른 메모리 공간에 저장된 별개의 값이다.
// 서로 영향을 주고 받지 않는다.
score = 100;

console.log(score, copy); // 100, 80
console.log(score === copy); // false
```

> 엄격하게 표현하자면 값이 전달된다는 표현은 부정확하다. 변수에는 **값**이 전달되는 게 아니라 **메모리 주소**가 전달되기 때문이다. 식별자는 값이 아니라 메모리 주소를 기억한다.

## 11.2. 객체

- 객체의 프로퍼티는 동적으로 추가되고 삭제될 수 있다.
    - 따라서 메모리 공간의 크기를 사전에 정해둘 수 없다.

### 자바스크립트 객체의 관리 방식

- 자바스크립트 객체는 프로퍼티 키를 인덱스로 사용하는 해시 테이블과 유사하다.
- 클래스 기반 객체지향언어에서는 사전에 정의된 클래스를 기반으로 인스턴스를 생성해, 인스턴스가 생성된 이후에는 프로퍼티나 메서드를 동적으로 변경할 수 없다.
- 반면 자바스크립트는 객체가 생성된 이후에도 프로퍼티와 메서드를 동적으로 변경할 수 있다.
    - 이는 성능면에서 객체의 생성과 프로퍼티 접근에 비용이 더 많이 드는 방식이다.
    - 성능상 비효율을 낮추기 위해 **히든 클래스**라는 방식을 사용한다.

> [!NOTE]
> **히든 클래스**
> - 명세(ECMAScript)에는 없는 개념이고, **엔진 구현 레벨에서의 최적화 메커니즘**이다.
> - 객체가 어떤 프로퍼티를 어떤 순서로 가지고 있는지 매핑해두는 역할
> - 모든 객체는 하나의 히든 클래스만을 참조한다.
> - 프로퍼티의 순서에 따라 다른 히든 클래스가 생성된다.

### 참조에 의한 전달

- 객체를 가리키는 변수를 다른 변수에 할당하면 원본의 참조 값이 복사되어 전달된다.
