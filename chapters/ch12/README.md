# 12. 함수

## 12.1. 함수란?

- 일련의 과정을 문으로 구현하고 코드 블록으로 감싸서 하나의 실행 단위로 정의한 것

## 12.2. 함수를 사용하는 이유

- 유지보수의 편의성
- 코드의 신뢰성
- 코드의 가독성

## 12.3. 함수 리터럴

- 자바스크립트의 함수는 **호출할 수 있는 객체다**
- 함수 리터럴의 구성 요소

    | 구성요소 | 설명 |
    | --- | --- |
    | 함수 이름 | • 함수 이름은 식별자다<br>• **함수 이름은 함수 몸체 내에서만 참조할 수 있는 식별자다**<br>• 생략할 수 있다 |
    | 매개변수 목록 | • 매개변수 순서에 따라 인수가 할당된다<br>• 함수 몸체 내에서 변수와 동일하게 취급된다 |
    | 함수 몸체 | • 호출됐을때 실행될 문의 모음 |

## 12.4. 함수 정의

> [!NOTE]
> 변수는 선언한다고 표현하지만 함수는 정의한다고 표현한다
> 함수 선언문이 평가되면 식별자가 암묵적으로 생성되고 함수 객체가 할당된다

### 12.4.1. 함수 선언문

```jsx
function add(x, y) {
  return x + y;
}
```

- 함수 선언문은 함수 이름을 생략할 수 없다
- 함수 선언문은 표현식이 아닌 문이다 (= 변수에 할당할 수 없다)
- 함수 선언문과 함수 리터럴은 형태가 동일하다
    - 기명 함수 리터럴은 중의적인 형태여서 코드의 문맥에 따라 해석이 달라질 수 있다
    - 기명 함수 리터럴을 단독으로 사용하면 함수 선언문으로 해석되고,
    변수에 할당하거나 피연산자로 사용하면 함수 리터럴로 해석된다
- 함수 이름은 함수 몸체 내에서만 참조할 수 있는 식별자다
    - 함수 외부에서 함수를 호출할 때 사용하는 식별자는 **자바스크립트가 암묵적으로 생성한 식별자다**
    - 함수는 함수 이름으로 호출하는 것이 아니라 함수 객체를 가리키는 식별자로 호출한다

    ```jsx
    function add (x, y) {
    	return x + y;
    }
    console.log(add(1,3)); // 4
    
    // 위 코드를 의사 코드로 표현하면 다음과 같다 ->
    
    var add = function add (x, y) {
    	return x + y;
    }
    console.log(add(1,3)); // 4
    ```

### 12.4.2. 함수 표현식

- 자바스크립트의 함수는 일급 객체다
- 함수 선언문은 "표현식이 아닌 문"이고 함수 표현식은 "표현식인 문"이다

### 12.4.3. 함수 생성 시점과 함수 호이스팅

```jsx
console.log(add); // add(x, y)
console.log(sub); // undefined

console.log(add(2, 5)); // 7
console.log(sub(2, 5)); // TypeError: sub is not a function

// 함수 선언문
function add(x, y) {
	return x + y;
}

// 함수 표현식
var sub = function(x, y) {
	return x - y;
}
```

- 함수 선언문으로 정의한 함수와 함수 표현식으로 정의한 함수의 생성 시점은 다르다
- 함수 선언문으로 정의한 함수는 런타임 이전에 함수 객체가 먼저 생성된다
    - 이를 함수 호이스팅이라고 한다
- 변수 할당문은 런타임에 평가된다
    - 함수 표현식으로 함수를 정의하면 함수 호이스팅이 아닌 변수 호이스팅이 발생한다
- 함수 호이스팅이 일어나는 함수 선언문 대신 함수 표현식이 더 권장된다.

- 다른 방식으로 생성한 함수와는 다르게 동작한다

## 12.5. 함수 호출

### 12.5.1. 매개변수와 인수

- 함수 호출시 필요한 값이 있다면 외부에서 매개변수를 통해 인수를 전달한다
- 매개변수는 함수를 정의할 때 생성되며, 변수와 동일하게 취급된다
- 매개변수의 스코프는 함수 내부다
- 인수가 할당되지 않은 매개변수는 undefined이며, 초과된 인수는 무시된다

### 12.5.2. 인수 확인

- 자바스크립트 함수는 매개변수와 인수의 개수가 일치하는지 확인하지 않는다
- 자바스크립트의 함수는 매개변수의 타입을 사전에 지정할 수 없다
- ES6에는 매개변수 기본값 문법이 추가됨

### 12.5.3. 매개변수의 최대 개수

- 함수는 하나의 역할만 하는게 좋으며, 여러가지 일을 하는 것은 바람직하지 않다
- 매개변수의 개수는 가급적 3개를 넘기지 않는 게 권장된다

### 12.5.4. 반환문

- 함수 호출은 표현식이다
- 반환문은 함수 실행을 중단하고 함수 몸체를 빠져나간다
- 반환문은 return 키워드 뒤에 오는 표현식을 평가해 반환한다

## 12.6. 참조에 의한 전달과 외부 상태의 변경

- 함수의 인수로 객체를 전달하면 함수 내부의 로직에 의해 객체 상태가 변경될 수 있다

## 12.7. 다양한 함수의 형태

### 12.7.1. 즉시 실행 함수

- 함수 정의와 동시에 즉시 호출되는 함수, 단 한번만 호출되며 다시 호출할 수 없다
- 즉시 실행 함수는 반드시 그룹 연산자 `(...)` 로 감싸야 한다
- 즉시 실행 함수도 일반 함수처럼 값을 반환할 수 있고 인수를 전달할 수도 있다

### 12.7.2. 재귀 함수

- 자기 자신을 호출하는 함수
- 무한 반복으로 인한 스택 오버플로우를 조심해야 한다

### 12.7.3. 중첩 함수

- 함수 내부에 정의된 함수, 스코프와 클로저에 깊은 관련이 있다

### 12.7.4. 콜백 함수

- 함수의 매개변수를 통해 다른 함수의 내부로 전달되는 함수
- 매개변수를 통해 외부에서 콜백함수를 전달받은 함수는 고차 함수라고 한다
- 콜백 함수는 고차 함수를 통해 호출되고, 인수를 전달 받을 수 있다

### 12.7.5. 순수 함수와 비순수 함수

- 순수 함수: 어떤 외부 상태에 의존하지도 않고 외부 상태를 변경하지도 않는 함수
- 비순수 함수: 어떤 외부 상태에 의존하거나 외부 상태를 변경하는 함수
