# 25. 클래스

## 25.1. 클래스는 프로토타입의 문법적 설탕인가?

- 자바스크립트는 프로토타입 기반 객체지향 언어다.
- 프로토타입 기반 객체지향 언어는 클래스가 필요 없다.
- 자바스크립트의 클래스 문법은 생성자 함수와 매우 유사하게 동작하지만 몇 가지 차이가 있다.

## 25.2. 클래스 정의

- class 키워드를 사용하여 정의한다.
- 클래스는 표현식으로 정의할 수 있다.
    
    ```jsx
    const Person = class {};
    const person = class Person {};
    ```
    
- 클래스는 함수며, 따라서 일급 객체다.
- 클래스는 0개 이상의 메서드를 가질 수 있으며, 생성자, 프로토타입 메서드, 정적 메서드를 가질 수 있다.

## 25.3. 클래스 호이스팅

- 클래스는 함수 선언문과 같이 런타임 이전에 먼저 평가되어 함수 객체를 생성한다.
    - 이때 프로토타입도 더불어 생성된다.
- 단, 클래스는 클래스 정의 이전에 참조될 수 없다.
    - 호이스팅은 일어나지만 let, const와 동일하게 호이스팅되며, 선언문 이전에는 TDZ에 빠지게 된다.

## 25.4. 인스턴스 생성

- 클래스는 new 연산자를 통해 인스턴스를 생성하며, new 연산자 없이 호출하는 것은 허용되지 않는다.
- 기명 함수 표현식과 동일하게 클래스명은 클래스 내부에서만 유효한 식별자다.

## 25.5. 메서드

### 25.5.1. constructor

- 인스턴스를 생성하고 초기화하기 위한 메서드
- 이름을 변경할 수 없다.
- constructor는 클래스 내에 최대 한개만 존재할 수 있다.
- constructor는 생략할 수 있다.
- constructor 내에 return 문을 넣으면 this가 아닌 return한 객체가 반환된다.

### 25.5.2. 프로토타입 메서드

- 클래스 내부에 정의한 메서드는 자동으로 프로토타입 메서드가 된다.

### 25.5.3. 정적 메서드

- 클래스 내부에 정의한 메서드에 static 키워드를 붙이면 정적 메서드가 된다.
- 정적 메서드는 클래스에 바인딩된다.
- 인스턴스는 프로토타입 체인 내에 클래스가 존재하지 않기 때문에 정적 메서드를 호출할 수 없다.

### 25.5.4. 정적 메서드와 프로토타입 메서드의 차이

- 정적 메서드는 인스턴스 프로퍼티를 참조할 수 없다.

### 25.5.5. 클래스에서 정의한 메서드의 특징

- function 키워드를 생략한다.
- 암묵적으로 strict mode로 실행된다.
- 열거 불가능 객체다.
- 비 생성자 함수다.

## 25.6. 클래스의 인스턴스 생성 과정

- 생성자 함수와 유사한 과정을 거친다.
1. 인스턴스 생성과 this 바인딩
2. 인스턴스 초기화
3. 인스턴스 반환

## 25.7. 프로퍼티

### 25.7.1. 인스턴스 프로퍼티

- 인스턴스 프로퍼티는 constructor 내부에서 정의해야 한다.
- constructor 내부에서 this에 추가한 프로퍼티는 언제나 인스턴스의 프로퍼티가 된다.
- 인스턴스 프로퍼티는 언제나 public하다.

### 25.7.2. 접근자 프로퍼티

- 자체적으로는 값을 갖지 않고 다른 데이터 프로퍼티의 값을 읽고 쓸 때 사용하는 접근자 함수로 구성된 프로퍼티다.

### 25.7.3. 클래스 필드 정의

- 클래스 필드란 클래스 기반 객체지향 언어에서 클래스가 생성할 인스턴스의 프로퍼티를 가리킨다.
- 자바스크립트는 constructor 내부에서만 인스턴스 프로퍼티를 정의하도록 했었지만,
현재는 ECMAScript에 클래스 필드 정의가 추가되었다.
    - https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Classes/Public_class_fields
    - https://v8.dev/features/class-fields
- 클래스 내부에서 this와 혼용해서 사용할 수 있다.
- 클래스 필드 정의와 초기 할당을 동시에 할 수 있다.

### 25.7.4. private 필드 정의

- private 필드는 변수명 앞에 `#`을 붙여야 한다.
- private 필드는 클래스 내부에서만 참조할 수 있다.
- 클래스 내부에 정의되지 않은 private 필드를 호출할 시 문법 오류가 발생한다.
- `delete`, 열거, `Object.*`, Proxy 등으로는 관찰하거나 조작할 수 없다.
- https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Classes/Private_elements

### 25.7.5. static 필드 정의

- static 메서드와 마찬가지로 static 필드를 정의할 수 있다.
- https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Classes/static

### 번외

- static 초기화 블럭
    - https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Classes/Static_initialization_blocks
    - 클래스 선언 내부에서 실행되는 초기화용 코드 블록으로, 클래스가 정의될 때 한 번 평가된다.
    - 단순한 `static` 필드 초기화보다 유연해서 `try...catch` 같은 구문을 쓰거나 여러 static 필드를 한 번에 설정하는 등의 복잡한 초기화 로직을 캡슐화할 수 있다.

## 25.8. 상속에 의한 클래스 확장

### 25.8.1. 클래스 상속과 생성자 함수 상속

- 프로토타입 기반 상속: 프로토타입 체인을 통해 다른 객체의 자산을 상속
    
    > "이 객체에 없으면 저 객체에서 찾아봐."
    > 
- 클래스 기반 상속: 기존 클래스를 상속받아 새로운 클래스를 확장하여 정의
    
    > "Dog는 Animal을 확장한 클래스야."
    > 
- 자바스크립트의 상속은 본질적으로 프로토타입 기반이지만, `class extends` 문법을 통해 클래스 기반 언어처럼 상속을 표현할 수도 있다.

### 25.8.2. extends 키워드

```jsx
class Base {}
class Derived extends Base {}
```

### 25.8.3. 동적 상속

- 클래스가 아닌 생성자 함수를 상속받아 확장할 수 있다.

### 25.8.4. 서브클래스의 constructor

- 서브클래스에서 constructor를 생략하면 암묵적으로 super를 호출하는 constructor가 정의된다.

### 25.8.5. super 키워드

- 함수처럼 호출하면 수퍼클래스의 constructor를 호출한다.
    - 서브클래스에서 constructor를 직접 정의한다면 super를 반드시 호출해야 한다.
    - super를 호출하기 전에는 this를 참조할 수 없다.
    - constructor 외부에서는 super를 호출할 수 없다.
- 참조하면 수퍼클래스의 메서드를 호출할 수 있다.

### 25.8.6. 상속 클래스의 인스턴스 생성 과정

- 자바스크립트 엔진은 수퍼클래스와 서브클래스를 구분하기 위한 내부 슬롯`[[ConstructorKind]]`을 갖는다.
    - 다른 클래스를 상속받지 않는 클래스의 슬롯 값: `base`
    - 다른 클래스를 상속받는 클래스의 슬롯 값: `derived`
1. 서브클래스의 super 호출
    - 서브클래스는 자신이 직접 인스턴스를 생성하지 않고 수퍼클래스에게 인스턴스 생성을 위임한다.
2. 수퍼클래스의 인스턴스 생성과 this 바인딩
    - constructor 내부 코드가 실행되기 전에 암묵적으로 빈 객체를 생성해 this 바인딩한다.
    - 수퍼클래스의 constructor에서도 new.target은 서브클래스를 가리킨다.
    - 인스턴스는 new.target이 가리키는 서브클래스가 생성한 것으로 처리된다.
3. 수퍼클래스의 인스턴스 초기화
4. 서브클래스 constructor로의 복귀와 this 바인딩
    - super가 반환한 인스턴스가 그대로 this에 바인딩된다.
    - super 호출 이전에는 인스턴스가 생성되지 않아 this를 참조할 수 없다.
5. 서브클래스의 인스턴스 초기화
6. 인스턴스 반환

### 25.8.7. 표준 빌트인 생성자 함수 확장

- extends를 통해 표준 빌트인 객체도 확장할 수 있다.

```jsx
class MyArray extends Array {
  uniq() {
    return [...new Set(this)];
  }
}
```

- `Array.prototype.map`, `filter`, `slice`처럼 새 배열을 반환하는 메서드는 기본적으로 현재 인스턴스의 생성자를 사용해 결과 객체를 만든다.
    - 따라서 MyArray 인스턴스에서 `map()`을 호출하면 결과도 일반 Array가 아니라 MyArray 인스턴스가 된다.
    - 이를 원하지 않는다면 `static get Symbol.species { return Array; }` 를 추가하면 된다.
    - `Symbol.species`는 복사본/파생 결과를 반환하는 내장 메서드가 **결과 객체의 생성자를 결정할 때 참고하는 정적 접근자**다.
