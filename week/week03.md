## 핵심 요약

- 생성자 함수를 사용하면 동일한 구조의 객체를 `new` 키워드로 간편하게 다수 생성할 수 있다.
- `new` 호출 시 엔진이 암묵적으로 빈 객체 생성 → this 바인딩 → 초기화 → this 반환 순서로 처리한다.
- 모든 함수는 callable이지만 constructor/non-constructor로 구분된다. 메서드(축약 표현)와 화살표 함수는 non-constructor다.
- 함수는 일급 객체로서 변수에 저장하거나, 인수로 전달하거나, 반환값으로 사용할 수 있다.
- 자바스크립트 상속은 **프로토타입 기반**으로 구현된다. 프로토타입과 생성자 함수는 언제나 쌍으로 존재한다.
- **프로토타입 체인**은 프로퍼티 검색 메커니즘, **스코프 체인**은 식별자 검색 메커니즘이다. 최상위는 `Object.prototype`.

## 토론 내용

### 자바스크립트는 왜 프로토타입을 선택했을까

- 클래스 기반 언어는 인스턴스 생성 전 클래스 정의가 고정되어 있지만, 자바스크립트는 런타임에 객체를 동적으로 확장할 수 있어야 했다.
- 프로토타입 기반 모델은 객체가 다른 객체를 직접 상속할 수 있어 유연한 구조를 만들기에 적합하다.
